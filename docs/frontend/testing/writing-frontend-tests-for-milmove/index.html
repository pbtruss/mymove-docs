Writing Frontend Tests for MilMove

This guide covers how MilMove writes tests using RTL. Perhaps you have stumbled across this guide, but are looking for generic strategies. That is covered in this wiki.

Common Mocks#

React Router Dom#

const mockPush = jest.fn();const mockGoBack = jest.fn();
jest.mock('react-router-dom', () => ({  ...jest.requireActual('react-router-dom'),  useLocation: () => ({    pathname: 'localhost:3000/',  }),  useHistory: () => ({    push: mockPush,    goBack: mockGoBack,  }),}));

What is this used for? A lot of pages have buttons that take the user to different places. We want to verify that users get redirected properly

Internal API#

jest.mock('services/internalApi', () => ({  ...jest.requireActual('services/internalApi'),  someApiCall: someFunction}));

What is this used for? A lot of components make service calls to the API. Being component tests, we don't actually want those to make calls to the service because that will make our tests brittle. Instead, we mock out the service calls that we need.
